{# always-visible verdict legend (glyph + token color, colorblind-safe) #}
Verdicts: ✕ reject ↑ recover ↔ drift ✓ agrees ? unverifiable
What each verdict means

reject — the current gate would reject a fit that was accepted (a suspect value already in state). recover — it now accepts a fit that was discarded (re-apply candidate). drift — both accept but the fitted number moved past tolerance. unverifiable — the replay couldn't decide. Every verdict is anchored to a specific gate (its sha256), so it reads as "disagrees with gate X", never "this fit is bad".
